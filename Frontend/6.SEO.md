## 1. Rendering Strategies & SEO

This is the most common interview topic. How you deliver code affects how bots "see" it.

- **CSR (Client-Side Rendering):** Think of a basic React app. The HTML is nearly empty, and JS builds the UI.
- **The Risk:** Google has a "second wave" of indexing. It crawls the HTML first, then comes back later (can be days) to run the JS. Your site might not be indexed correctly for a while.

- **SSR (Server-Side Rendering):** Used in Next.js (`getServerSideProps`). The server sends a fully populated HTML file.
- **The Pro:** Bots see the content immediately. Great for dynamic content that changes often (e.g., news).

- **SSG (Static Site Generation):** Used in Next.js (`getStaticProps`). Pages are built at build-time.
- **The Pro:** The fastest possible performance. Best for blogs or documentation.

## 2. Semantic HTML

Don't just use `<div>` for everything. Browsers and crawlers use tags to understand the **hierarchy** and **importance** of content.

- **`<main>`**: The primary content of the body.
- **`<nav>`**: Navigation links.
- **`<article>` & `<section>**`: Define independent or grouped content.
- **Heading Hierarchy:** Use only **one `<h1>**` per page. Follow a logical order (`<h1>`->`<h2>`->`<h3>`). Don't skip levels for styling purposes.
- **`<aside>`**: For content indirectly related to the main piece (like sidebars).

---

## 3. Metadata & Social Crawlers

Meta tags live in the `<head>` and tell the bot what the page is about.

- **Title & Description:** These appear in search results. Keep titles under 60 characters and descriptions under 160.
- **Open Graph (OG) & Twitter Cards:** These control how your link looks when shared on social media.

```html
<meta property="og:title" content="Expert Front-End Tips" />
<meta property="og:image" content="https://site.com/thumbnail.jpg" />
```

- **Canonical Tags:** If you have the same content on two URLs (e.g., `/products?id=1` and `/products/shoes`), use `<link rel="canonical" href="...">` to tell Google which one is the "official" version to avoid duplicate content penalties.

## 4. Structured Data (JSON-LD)

Structured data helps you get **Rich Snippets** (star ratings, prices, or FAQ dropdowns appearing directly on Google).

You implement this using a `<script type="application/ld+json">` block. It helps the bot understand that "This isn't just text; it's a **Product** with a **Price** of $50."

```html
<script type="application/ld+json">
  {
    "@context": "http://schema.org",
    "@type": "Product",
    "name": "Product Name",
    "description": "Product Description",
    "image": "https://site.com/image.jpg",
    "brand": "Brand Name",
    "aggregateRating": {
      "@type": "AggregateRating",
      "ratingValue": "4.5",
      "reviewCount": "123"
    },
    "offers": {
      "@type": "Offer",
      "priceCurrency": "USD",
      "price": "50.00",
      "availability": "InStock"
    }
  }
</script>
```

## 5. Web Vitals & Mobile-First

Google explicitly uses **Page Experience** as a ranking factor.

- **Mobile-First Indexing:** Google crawls the mobile version of your site first. If your mobile UI is hidden or broken, your desktop ranking will drop.
- **Core Web Vitals:** (LCP, INP, CLS). A slow or "jumpy" site is penalized.
- **HTTPS:** A secure site is a requirement for modern SEO.

---

## 6. Accessibility (A11y) is SEO

Accessibility and SEO are two sides of the same coin.

- **`alt` attributes on images:** Bots can't "see" images; they read the `alt` text.
- **Descriptive links:** Avoid "Click here." Use "Download the SEO Guide PDF." This tells the bot what the linked page is about.

---

## 7. Robots.txt

The `robots.txt` file is a text file that tells bots which pages to crawl and which to avoid. It's a **technical requirement** for SEO.

```html
User-agent: * Disallow: /admin Allow: /public
```

---

## 8. Sitemap.xml

The `sitemap.xml` file is an XML file that lists all the pages on your site. It helps the bot understand the structure of your site and crawl it more efficiently.

```html
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <url>
    <loc>https://site.com/</loc>
    <lastmod>2022-01-01</lastmod>
    <changefreq>daily</changefreq>
    <priority>1.0</priority>
  </url>
</urlset>
```

---

## 9. Robots vs Sitemap vs JSON-LD

- **Robots.txt:** Telling bots which pages to crawl and which to avoid.
- **Sitemap.xml:** Telling bots which pages exist and how often they change.
- **JSON-LD:** Telling bots what the page is about.

### Interview "Cheat Sheet" Questions:

- **"How does Next.js help with SEO?"**
- _Answer:_ By providing SSR/SSG to ensure the bot receives full HTML, and the `<Head>` component for managing metadata per page.

- **"What is the 'Double Indexing' problem in SPAs?"**
- _Answer:_ It's when Google crawls the empty shell of a JS app first and fails to see the content until the second pass after rendering the JS.

- **"How do you prevent a page from being indexed?"**
- _Answer:_ Use `<meta name="robots" content="noindex">` or the `robots.txt` file.
