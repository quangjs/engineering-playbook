# 1. CSS Fundamentals (Core Knowledge)

## 1.1 CSS Architecture & Organization

### Common Approaches

- **Global CSS** – traditional stylesheet
- **CSS Modules** – locally scoped styles
- **CSS-in-JS** – styles written inside JS
- **Utility-first CSS** – atomic classes (e.g., Tailwind)
- **BEM methodology** – Block\_\_Element--Modifier naming

### BEM Example

```css
.card {
}
.card__title {
}
.card--highlighted {
}
```

### What Interviewers Care About

- How to avoid style conflicts
- How to scale CSS in large teams
- How to prevent specificity wars
- Maintainability and reusability

---

## 1.2 CSS Specificity & Cascade

You must understand:

- Inline styles > ID > Class > Element
- `!important` overrides everything (avoid using it)
- Later rules override earlier rules (if same specificity)

Interview Question:

> Why is my style not applying?

Answer often relates to:

- Specificity
- Cascade order
- Shadow DOM
- CSS Modules scoping

---

## 1.3 Layout Systems

### Flexbox

Use for:

- 1-dimensional layout
- Aligning items
- Centering content

Key properties:

- `display: flex`
- `justify-content`
- `align-items`
- `flex: 1`

### Grid

Use for:

- 2-dimensional layouts
- Complex dashboards
- Layout templates

Key properties:

- `grid-template-columns`
- `grid-template-rows`
- `gap`
- `grid-area`

Interview tip:

> Flexbox = components
> Grid = page layout

---

## 2.2 Media Queries

```css
@media (max-width: 768px) {
  .sidebar {
    display: none;
  }
}
```

Common breakpoints:

- 576px (mobile)
- 768px (tablet)
- 1024px (desktop)
- 1280px+ (large desktop)

---

## 2.3 Mobile-First vs Desktop-First

### Mobile-First (Recommended)

```css
.button {
  font-size: 14px;
}

@media (min-width: 768px) {
  .button {
    font-size: 16px;
  }
}
```

Advantages:

- Better performance
- Cleaner scaling
- Easier progressive enhancement

---

## 2.4 Responsive Units

- `px` → fixed
- `%` → relative to parent
- `rem` → relative to root
- `em` → relative to parent font-size
- `vh`, `vw` → viewport-based

Interview Question:

> Why prefer `rem` over `px`?

Answer:

- Accessibility (user font scaling)
- More scalable design system

- Large ecosystem

- Enterprise-focused
- Strong form components
- Used in admin systems

# 5. CSS-in-JS vs Traditional CSS

## CSS-in-JS (e.g., styled-components)

Pros:

- Scoped styles
- Dynamic styling
- Theme support

Cons:

- Runtime cost
- Larger bundle

---

## CSS Modules

Pros:

- Scoped
- No runtime overhead

Cons:

- Less dynamic

---

## Tailwind

Pros:

- Fast dev
- No naming problem
- Predictable

Cons:

- HTML looks messy
- Harder to enforce strict design rules

Interview Answer Strategy:

- Explain trade-offs
- Mention performance
- Mention team size and project scale
