## 1. Routing (The Navigation Layer)

Since React is a Single Page Application (SPA) library, routing is handled on the client side to prevent full page reloads.

- **React Router (Web):** The industry standard. It uses a declarative approach to map URL paths to components.
- _Key concepts:_ `BrowserRouter`, `Routes`, `Route`, `Link`, and hooks like `useNavigate` and `useParams`.

- **TanStack Router:** A newer, **fully type-safe** router that is gaining massive popularity for its advanced data-loading and caching capabilities.
- **Framework Routing:** If using **Next.js** or **Remix**, routing is file-system based (e.g., creating a file at `app/about/page.tsx` automatically creates the `/about` route).

---

## 2. Data Fetching & Server State

Modern React development has moved away from fetching data inside `useEffect`. We now use specialized "Server State" libraries.

- **TanStack Query (React Query):** The gold standard. It handles:
- **Caching:** Avoiding redundant API calls.
- **Auto-refetching:** Updating data when the window is refocused.
- **Pagination & Infinite Scroll:** Built-in support.

- **SWR (by Vercel):** A simpler, lightweight alternative to React Query based on the "Stale-While-Revalidate" strategy.
- **Axios:** Still the preferred library for making HTTP requests due to its easy interceptors and default JSON handling compared to the native `fetch`.

## 3. Styling & UI Components

How you manage CSS in React impacts bundle size and developer experience (DX).

| Methodology        | Library Examples                             | Pros                                                                   |
| ------------------ | -------------------------------------------- | ---------------------------------------------------------------------- |
| **Utility-First**  | **Tailwind CSS**                             | Fast development, no "CSS files," tiny production bundle.              |
| **CSS-in-JS**      | **Styled Components**, **Emotion**           | Scoped styles, dynamic props, easy to maintain logic.                  |
| **Component Libs** | **MUI**, **Ant Design**, **Chakra UI**       | Pre-built, accessible components (Inputs, Modals, Tables).             |
| **Headless UI**    | **Radix UI**, **Headless UI**, **Shadcn/ui** | Provides logic/accessibility; you provide the CSS (The current trend). |

---

## 4. Forms & Validation

Handling form state manually with `useState` for 20 inputs is inefficient.

- **React Hook Form:** The most popular choice. It uses `uncontrolled` inputs under the hood to minimize re-renders, making it extremely performant.
- **Zod / Yup:** Schema validation libraries. You define the "shape" of your data, and they automatically validate the form inputs and provide TypeScript types.

---

## 5. State Management (The "Big Three")

As discussed previously, choosing the right tool depends on the scope of the data.

1. **Context API:** Built-in. Best for static/low-frequency data (Theme, User Auth).
2. **Zustand:** The modern go-to. Extremely simple, hook-based, and performant.
3. **Redux Toolkit:** The enterprise choice. Great for complex, predictable state transitions and powerful debugging tools.

---

## 6. Testing (Ensuring Quality)

- **Vitest / Jest:** The test runner for unit tests.
- **React Testing Library (RTL):** Encourages testing components the way a user interacts with them (e.g., searching for a button by text rather than by a CSS class).
- **Playwright / Cypress:** For End-to-End (E2E) testing—simulating a real user in a real browser.

---

## 7. Build Tools & Frameworks

- **Vite:** The fastest modern build tool for standard React apps (replaces Create React App).
- **Next.js:** The most popular React framework. It adds SSR, SSG, ISR, and Server Components out of the box.
- **Remix:** A framework focused on web standards (forms, HTTP caching) and smooth data transitions.
