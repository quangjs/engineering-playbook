## 1. The React Runtime & Rendering

Understanding how React works under the hood is critical for explaining "the why" behind your code.

- **React Lifecycle:**
  - **Mounting:** The component is created and inserted into the DOM.
  - **Updating:** The component is re-rendered due to changes in props or state.
  - **Unmounting:** The component is removed from the DOM.
- **Virtual DOM & Reconciliation:** React creates a tree of objects representing the UI. When state changes, it creates a new tree and "diffs" it against the old one to update only what is necessary.
- **The Fiber Engine:** The modern reconciliation engine that allows React to pause, abandon, or reuse rendering work. This enables **Concurrent Features**.
- **JSX:** A syntax extension that transpiles to `React.createElement()` calls.
- **Keys in Lists:** Why `key` is required—it acts as a stable identity for elements so React doesn't re-mount components unnecessarily during list reorders.

## 2. State Management & Data Flow

Managing how data moves through an application is the most common source of complexity.

- **Unidirectional Data Flow:** Data moves down (props), actions move up (callbacks).
- **Local vs. Global State:** \* **Local:** `useState`, `useReducer`.
- **Global:** Context API (for low-frequency updates like Theme/Auth) or external libraries like **Redux Toolkit**, **Zustand**, or **Recoil**.

- **Prop Drilling:** The challenge of passing data through many layers; solved by Context or Component Composition.
- **Server State:** Moving away from `useEffect` for data fetching. Using **TanStack Query (React Query)** or **SWR** for caching, synchronization, and optimistic updates.

---

## 3. Hooks API (The Standard)

Hooks are the primary way to use React features in functional components.

- **State Hooks:** `useState` for simple values, `useReducer` for complex state transitions.
- **Effect Hooks:** `useEffect` for side effects. Understanding the **cleanup function** and the **dependency array** is vital to avoid memory leaks and infinite loops.
- **Ref Hooks:** `useRef` for persisting values without re-renders or accessing DOM nodes directly.
- **Performance Hooks:** `useMemo` (memoize values) and `useCallback` (memoize functions). Use these to prevent unnecessary downstream re-renders.

## 4. Component Patterns & Reusability

Writing "DRY" (Don't Repeat Yourself) code in React requires specific patterns.

- **Component Composition:** Passing components as `children` to avoid deep prop drilling.

```javascript
function Modal({ children }) {
  return <div className="modal">{children}</div>;
}
```

- **Custom Hooks:** Encapsulating complex logic (e.g., `useAuth`, `useWindowSize`) to be shared across components.
- **Controlled vs. Uncontrolled:** Controlled components have their value driven by React state; uncontrolled rely on the DOM (via `refs`).

```javascript
// Controlled component
function ControlledInput() {
  const [value, setValue] = useState("");
  return (
    <input
      type="text"
      value={value}
      onChange={(e) => setValue(e.target.value)}
    />
  );
}
// Uncontrolled component
function UncontrolledInput() {
  const inputRef = useRef(null);
  return <input type="text" ref={inputRef} />;
}
```

- **Higher-Order Components (HOC) & Render Props:** Older but still relevant patterns for sharing logic.
- **Compound Components:** A pattern where components work together to form a unit (e.g., `<Tabs>` and `<Tab>`).

## 5. Performance Optimization

Interviewers will often present a "slow app" and ask how you would fix it.

- **Code Splitting:** Using `React.lazy` and `Suspense` to load parts of the application only when needed (reducing initial bundle size).
- **Memoization:** Using `React.memo` to prevent a component from re-rendering if its props haven't changed.
- **Windowing/Virtualization:** Using libraries like `react-window` to render only the visible items in a massive list.
- **Profiling:** Using the React DevTools Profiler to identify "bottleneck" components that render too often.

---

## 6. Ecosystem & Advanced Rendering

React rarely exists in a vacuum.

- **Error Boundaries:** Using `componentDidCatch` or `getDerivedStateFromError` to catch JavaScript errors in the child component tree and display a fallback UI.
- **Hydration & SSR/SSG:** Understanding how **Next.js** or **Remix** differ from standard Create React App (CSR) by pre-rendering HTML on the server.
- **Testing:** \* **Unit/Integration:** React Testing Library (focuses on user behavior) and Jest.
- **E2E:** Cypress or Playwright.

### Comparison: When to use which "Memo"?

| Tool              | Purpose                       | Use Case                                               |
| ----------------- | ----------------------------- | ------------------------------------------------------ |
| **`React.memo`**  | Prevents Component Re-render  | Wraps a component that receives the same props often.  |
| **`useMemo`**     | Prevents Re-calculation       | Wraps an expensive math operation or object creation.  |
| **`useCallback`** | Prevents Function Re-creation | Wraps a function passed as a prop to a memoized child. |
